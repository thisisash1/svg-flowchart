# 레이아웃 가이드

## 방향 선택 원칙

| 플로우 유형 | 방향 | 캔버스 |
|------------|------|--------|
| 사용자 여정, 결제, 로그인, 송금, 회원가입 | **세로** (top→bottom) | 1800 × 3200~4000 |
| 시스템 아키텍처, API 흐름, 인프라 | **가로** (left→right) | 3200~4800 × 1400 |
| 부서별 업무, 역할 분리, 다중 액터 | **스윔레인** (가로 레인 + 세로 진행) | 2400~3600 × 1800~2400 |
| 의사결정 트리, 케이스 분기 확장 | **혼합** (세로 기둥 + 가로 분기) | 2400 × 2400~3200 |

## 높이 계산 공식 (세로형 기준)

```
총 높이 = 40(상단 여백)
        + 72(타이틀) + 16(간격)
        + 48(범례) + 40(간격)
        + [메인 플로우 노드 수 × 120]
        + [다이아몬드 수 × 80]
        + [분기 END 노드 추가 높이]
        + 80(섹션 헤더) + [예외 케이스 노드 수 × 100] + 80(예외 여백)
        + 100(컴포넌트 배지 구역, 있을 경우)
        + 60(푸터) + 60(하단 여백)
```

## 패턴 A: 세로형 (Vertical) — 기본

선형 사용자 여정, 결제/로그인/회원가입 플로우에 사용.

```
캔버스: width=1800, height=가변
컬럼 구조:
  x=60~340    : 왼쪽 주석 / 예외 케이스
  x=340~1460  : 메인 플로우 (cx=900 중심)
  x=1460~1740 : 오른쪽 주석 / 배지

메인 노드:  x=660, width=480 → cx=900
넓은 노드:  x=560, width=680 → cx=900
흐름 방향: top → bottom
```

수직 간격:
- 스텝 노드 80px + 화살표 40px → 다음 Y = 전Y + 120
- 스텝 노드 56px + 화살표 40px → 다음 Y = 전Y + 96
- 다이아몬드 72px + 화살표 44px → 다음 Y = 전Y + 116
- 섹션 헤더 60px + 여백 20px → 다음 Y = 전Y + 80

우회 경유점(waypoint) 범위:
- 오른쪽 우회: wx = `1220~1460`
- 왼쪽 우회: wx = `340~580`

## 패턴 B: 가로형 (Horizontal) — 시스템/API 플로우

```
캔버스: width=가변(3200~4800), height=1400
레이어 구조:
  y=220 : 레이어 1 (클라이언트 / 프론트엔드)
  y=500 : 레이어 2 (API 서버 / 백엔드)
  y=780 : 레이어 3 (DB / 외부 서비스)

메인 노드: y=660, height=80 → cy=700
흐름 방향: left → right
노드 간 수평 간격: 노드 width + 80px
```

## 패턴 C: 스윔레인 (Swimlane) — 역할 분리

```
캔버스: width=가변(2400~3600), height=가변(1800~2400)
레인 라벨 영역: x=60, width=160 (고정)
레인 내용 영역: x=220 ~ (width-60)

레인별 y 범위 (레인 높이 = 260px 기준):
  레인 1: y=200~460  (예: 사용자)
  레인 2: y=460~720  (예: 프론트엔드)
  레인 3: y=720~980  (예: 백엔드 서버)
  레인 4: y=980~1240 (예: 외부 시스템)

흐름 방향: left → right (레인 내부)
레인 간 연결: 수직 화살표 (레인 경계 교차)
```

레인 라벨 박스:
```svg
<rect x="60" y="{LANE_Y}" width="160" height="260" rx="8"
  fill="{LANE_COLOR}" stroke="{LANE_STROKE}" stroke-width="1.5"/>
<!-- 세로 텍스트 -->
<text transform="rotate(-90, 140, {LANE_Y+130})" x="140" y="{LANE_Y+130}"
  font-size="12" font-weight="700" fill="{TEXT_COLOR}" text-anchor="middle">레인명</text>
```

레인 구분선:
```svg
<line x1="60" y1="{LANE_Y}" x2="{WIDTH-60}" y2="{LANE_Y}"
  stroke="#E2E8F0" stroke-width="1" stroke-dasharray="4,4"/>
```

## 패턴 D: 혼합형 (Mixed) — 의사결정 트리 / 분기 확장

```
캔버스: width=2400, height=가변(2400~3200)
메인 줄기: cx=1200 (가로 중앙)
1단계 분기: cx=1200 → 좌(cx=700) / 우(cx=1700)
2단계 분기: 좌(cx=700) → 좌좌(cx=400) / 좌우(cx=1000)
```
